Создайте логическую модель базы данных для системы управления складом.
Система управления складом предназначена для учета товаров на складе, контроля их движения и отслеживания запасов. Она позволяет добавлять товары на склад, проводить инвентаризацию, отслеживать перемещение товаров между складами и продавать товары.
Логическая модель базы данных должна включать следующие элементы:
1. Таблица "Товары" с полями:
   - Идентификатор товара (ID)
   - Название товара
   - Описание товара
   - Единица измерения товара
   - Цена товара
   - Количество товара на складе
   - Дата добавления товара на склад
2. Таблица "Склады" с полями:
   - Идентификатор склада (ID)
   - Название склада
   - Адрес склада
3. Таблица "Поставщики" с полями:
   - Идентификатор поставщика (ID)
   - Название поставщика
   - Адрес поставщика
4. Таблица "Заказы" с полями:
   - Идентификатор заказа (ID)
   - Дата заказа
   - Идентификатор поставщика (ID)
   - Идентификатор склада (ID)
   - Название товара
   - Количество товара
5. Таблица "Перемещения" с полями:
   - Идентификатор перемещения (ID)
   - Дата перемещения
   - Идентификатор склада (ID)
   - Идентификатор товара (ID)
   - Количество товара
   - Идентификатор склада назначения (ID)
При создании логической модели базы данных необходимо учесть следующие требования:
1. Каждый товар должен иметь уникальное название и описание.
2. Каждый товар должен быть связан с единицей измерения и ценой.
3. Каждый товар должен иметь информацию о количестве на складе и дате добавления на склад.
4. Каждый склад должен иметь уникальное название и адрес.
5. Каждый заказ должен быть связан с поставщиком и складом.
6. Каждый заказ должен содержать информацию о названии товара и количестве.
7. Каждый заказ должен быть связан с перемещением товара на склад.
8. Каждый перемещение должен содержать информацию о количестве товара, складе назначения и дате перемещения.
